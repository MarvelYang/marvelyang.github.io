# bind

bind MDN 描述

bind() 函数会创建一个新绑定函数（bound function，BF）。绑定函数是一个 exotic function object（怪异函数对象，ECMAScript 2015 中的术语），它包装了原函数对象。调用绑定函数通常会导致执行包装函数。 绑定函数具有以下内部属性：

  • [[BoundTargetFunction]] - 包装的函数对象
  • [[BoundThis]] - 在调用包装函数时始终作为 this 值传递的值
  • [[BoundArguments]] - 列表，在对包装函数做任何调用都会优先用列表元素填充参数列表
  • [[Call]] - 执行与此对象关联的代码。通过函数调用表达式调用。内部方法的参数是一个 this 值和一个包含通过调用表达式传递给函数的参数的列表

当调用绑定函数时，它调用[[BoundTargetFunction]]上的内部方法[[Call]]，就像这样 Call(boundThis, args)。其中，boundThis 是[[BoundThis]]，args 是[[BoundArguments]]加上通过函数调用传入的参数列表。

绑定函数也可以使用new运算符构造，它会表现为目标函数已经被构建完毕了似的。提供的 this 值会被忽略，但前置参数仍会提供给模拟函数。

思路

这里利用了call方法（当然也可以用apply）实现，如果不用这两个方法的话，需要先手动实现这两个方法，这个以后再做进一步研究。

要实现bind函数，先来看下思路：

首先, 方法可以接收一个参数作为绑定上下文对象：

    // 基本架构

Function.prototype.myBind = function(context) {
    let fun = this; // 这里this就是需要绑定的方法
    function boundFun() {
        // 在这里执行原函数方法，利用 call 方法，改变其上下文
        return fun.call(context);
    }
    return boundFun;
};

emmmmmm……看起来翻版还有待改善，那么问题出在哪里呢？

首先我们要明白new操作符做了什么，你可以点击MDN 链接查看详细说明

这里简单说一下：

  1. 创建一个新的空对象{}
  2. 把新建对象的_proto_属性指向构造函数的 prototype 属性
  3. 把 this 指针指向新创建的对象（就是把上下文设置为新对象）
  4. 返回这个新对象

那么我们就可以从中发现一个问题，我们自己实现myBind函数，把上下文绑定了context，所以new操作符在上面第 3 步的时候的指向后，此时this已经被指向隐式创建的新对象的上下文，但是在后续执行call方法的时候，传递的依然是我们之前的context，所以changeNameMyBind改变的依然是obj，然后this.name = 'changeName'并没有给创建的新对象添加name属性，所以返回的对象的name属性也就为空。

那么该如何改进代码呢，很简单，判断一下boundFun的this是否存在于boundFun的原型链上，为什么这么判断呢？

因为new操作符在执行时，把this指向了隐式创建的新对象的上下文

那么就是说，根据this到底是否有改变，就可以知道是否应该更改call的上下文参数context

判断方法也简单，使用instanceof

Function.prototype.myBind = function(context) {
    let fun = this; // 这里this就是需要绑定的方法
    function boundFun() {
        let self = this instanceof boundFun ? this : context; // 判断 this 上下文
        // 在这里执行原函数方法，利用 call 方法，改变其上下文
        return fun.call(self);
    }
    return boundFun;
};

结果也正确了，剩下我们再处理一下接收后面传递的参数即可

// 手动实现 bind 函数最终结果
Function.prototype.myBind = function(context, ...args) {
    let fun = this; // 这里this就是需要绑定的方法
    function boundFun(...argsBound) {
        let self = this instanceof boundFun ? this : context;
        // 在这里执行原函数方法，利用 call 方法，改变其上下文
        return fun.call(self, ...args, ...argsBound);
    }
    return boundFun;
};
